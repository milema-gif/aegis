  <files>lib/aegis-checkpoint.sh</files>
  <behavior>
    - write_checkpoint(stage, phase, content): creates .aegis/checkpoints/{stage}-phase-{phase}.md with timestamp header, rejects >375 words, uses atomic tmp+mv
    - read_checkpoint(stage, phase): returns file content or empty string
    - list_checkpoints(): returns checkpoint paths sorted by mtime (newest last)
    - assemble_context_window(current_stage, count): reads last `count` checkpoints, formats each with "---" separator, returns concatenated block or empty string
  </behavior>
  <action>
    Create `lib/aegis-checkpoint.sh` following existing Aegis library conventions:
    - Shebang: `#!/usr/bin/env bash`
    - Header comment: "Aegis Pipeline -- Checkpoint library"
    - `set -euo pipefail`
    - Use `AEGIS_DIR="${AEGIS_DIR:-.aegis}"` (same as aegis-state.sh)
    - `CHECKPOINT_DIR="${AEGIS_DIR}/checkpoints"`

    **write_checkpoint(stage, phase, content):**
    - Validate: `stage` and `phase` required (use `${1:?}` pattern)
    - Count words: `echo "$content" | wc -w` — if >375, print error to stderr, return 1
    - Add timestamp header: `## Checkpoint: ${stage} -- Phase ${phase} -- $(date -u +"%Y-%m-%dT%H:%M:%SZ")`
    - Prepend header to content
    - `mkdir -p "$CHECKPOINT_DIR"`
    - Write to `${CHECKPOINT_DIR}/${stage}-phase-${phase}.md.tmp.$$` then `mv -f` to final path
    - Return 0 on success

    **read_checkpoint(stage, phase):**
    - Build path: `${CHECKPOINT_DIR}/${stage}-phase-${phase}.md`
    - If file exists: `cat` it, return 0
    - If not: return 0 (empty output, non-blocking)

    **list_checkpoints():**
    - If `$CHECKPOINT_DIR` does not exist or is empty: return 0 (empty output)
    - `ls -1t "$CHECKPOINT_DIR"/*.md 2>/dev/null | tac` (oldest first, newest last — tac reverses ls -1t)

    **assemble_context_window(current_stage, count):**
    - Default count=3 if not provided
    - Call list_checkpoints, take last `count` entries (tail -n "$count")
    - For each file: read content, append to output with `---` separator
    - If no checkpoints: return 0 with empty output
    - Wrap in `## Prior Stage Context` header

    All functions: never `exit` — always `return`. Callers use `|| true` for non-blocking behavior.
  </action>
