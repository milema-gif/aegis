    - verify_deploy_scope(roadmap_path): reads ROADMAP.md, checks all phases in current milestone are [x], returns "pass" or "fail:{details}"
    - verify_rollback_tag(): calls git tag -l 'aegis/*', returns "pass:{latest_tag}" or "fail:no-tag"
    - snapshot_running_state(): captures docker ps + pm2 jlist into .aegis/snapshots/pre-deploy-{timestamp}.json, returns snapshot path
    - run_preflight(project_name): runs all 4 checks + working tree check, displays PREFLIGHT banner, returns "pass" or "blocked:{reasons}"

  <action>
    Create `lib/aegis-preflight.sh` following existing Aegis library conventions:
    - Shebang: `#!/usr/bin/env bash`
    - Header comment: "Aegis Pipeline -- Deploy preflight guard"
    - `set -euo pipefail`
    - Use `AEGIS_DIR="${AEGIS_DIR:-.aegis}"` (same as other libs)
    - Source aegis-state.sh for STAGES array and state reading functions

    **verify_state_position():**
    - The 8 prior stages are: intake, research, roadmap, phase-plan, execute, verify, test-gate, advance (indices 0-7)
    - For each stage in STAGES[0..7]: call `read_stage_status "$stage"` and check it equals "completed"
    - If any is NOT completed: echo "fail:$stage_name" and return 1
    - If all pass: echo "pass" and return 0

    **verify_deploy_scope(roadmap_path):**
    - Default roadmap_path to ".planning/ROADMAP.md" if not provided
    - Read file, find lines matching `- [ ]` (incomplete phases) under the current milestone section
    - If any incomplete phase found: echo "fail:incomplete phases found" and return 1
    - If all `- [x]`: echo "pass" and return 0
    - Handle missing file: echo "fail:roadmap not found" and return 1

    **verify_rollback_tag():**
    - Run `git tag -l 'aegis/*' --sort=version:refname 2>/dev/null`
    - If empty: echo "fail:no-tag" and return 1
    - If tags exist: echo "pass:$(last tag)" and return 0

    **snapshot_running_state():**
    - Create snapshot dir: `mkdir -p "$AEGIS_DIR/snapshots"`
    - Build JSON via python3 with:
      - timestamp (UTC ISO)
      - git_head: `git rev-parse HEAD 2>/dev/null`
      - docker: array from `docker ps --format json` (or empty if docker unavailable)
      - pm2: array from `pm2 jlist` (or empty if pm2 unavailable/no processes)
      - working_tree_clean: boolean from `git status --porcelain`
    - Write to `.aegis/snapshots/pre-deploy-{timestamp}.json` using atomic tmp+mv
    - Echo the snapshot path and return 0

    **run_preflight(project_name):**
    - Run all 4 verifications + working tree check (`git status --porcelain`)
    - Collect results into a formatted PREFLIGHT CHECK banner using echo (not show_checkpoint -- the banner format is custom for preflight)
    - Format:
      ```
      DEPLOY PREFLIGHT CHECK
      ======================
      [PASS] State position:  All 8 prior stages completed
      [PASS] Deploy scope:    All roadmap phases complete
      [PASS] Rollback tag:    aegis/phase-N-name
      [PASS] Clean tree:      No uncommitted changes
      [PASS] State snapshot:  .aegis/snapshots/pre-deploy-xxx.json
      ```
    - If ALL pass: echo "pass" to stdout, return 0
    - If ANY fail: echo "blocked:{first_failure_reason}", return 1

    All functions: never `exit` -- always `return`. Use `|| true` pattern where needed for non-blocking operations within functions.
  </action>
