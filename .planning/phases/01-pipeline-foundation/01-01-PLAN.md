  </behavior>
  <action>
    1. Create `references/state-transitions.md` — the canonical 9-stage transition table as a reference document. Include: stage name, index (0-8), description, next stage on success, and note that "advance" is the only non-linear transition (loops to phase-plan or advances to deploy). This is documentation, not code — the orchestrator and bash helpers reference it.

    2. Create `templates/pipeline-state.json` — the initial state file template following the schema from research:
       - version: 1, project: "{{PROJECT_NAME}}", pipeline_id: "{{PIPELINE_ID}}"
       - current_stage: "intake", current_stage_index: 0
       - stages array with all 9 stages (status "pending" except intake which is "active")
       - integrations object (placeholder, filled at runtime by aegis-detect.sh)
       - config object: auto_advance: false, yolo_mode: false

    3. Create `lib/aegis-state.sh` — sourced shell library with these functions:
       - `AEGIS_DIR` variable (defaults to `.aegis` relative to project root)
       - `STAGES` array: ("intake" "research" "roadmap" "phase-plan" "execute" "verify" "test-gate" "advance" "deploy")
       - `init_state(project_name)`: copies template, replaces placeholders (project name, generates pipeline_id via uuidgen or date fallback), sets timestamps, creates .aegis/ dir
       - `read_current_stage()`: reads state.current.json, extracts current_stage using python3 json module, prints it
       - `get_stage_index(stage_name)`: returns the numeric index of a stage name
       - `advance_stage(remaining_phases)`: reads current state, determines next stage (index+1 for normal, index 3 for advance-loop, index 8 for advance-to-deploy), validates transition, calls journal_transition, then write_state. The `remaining_phases` param is only used when current stage is "advance" — pass count of remaining phases (0 means go to deploy).
       - `journal_transition(from, to, result, metadata)`: constructs JSON line with timestamp, appends to state.history.jsonl
       - `write_state(json_content)`: writes to state.current.json.tmp.$$, then mv -f over original (atomic)
       - `recover_state()`: checks if state.current.json is valid JSON (python3). If corrupt, reads last line of state.history.jsonl, rebuilds minimal state from journal. If journal also missing, returns error code 1.
       - Use `set -euo pipefail` at top. All JSON manipulation via python3 (not jq — research says both work but python3 is more reliable for complex operations).

    4. Create test scripts (write tests FIRST, then verify they fail, then ensure lib makes them pass):
       - `tests/test-state-transitions.sh`: Tests init_state, read_current_stage, advance_stage (normal + advance loop + advance-to-deploy + invalid). Uses a temp directory for isolation. Each test: setup, action, assertion (compare expected vs actual), cleanup.
       - `tests/test-journaled-state.sh`: Tests journal_transition (appends valid JSONL), write_state (atomic — verify no .tmp files remain), recover_state (corrupt state file recovery, missing state file recovery, empty journal handling).
       - Both scripts: print "PASS: test_name" or "FAIL: test_name" per test, exit 0 if all pass, exit 1 if any fail.
  </action>

Returns JSON object: `{"engram":{"available":bool,"fallback":"..."},"sparrow":{"available":bool,"fallback":"..."},"codex":{"available":bool,"gated":true,"note":"user-explicit only"}}`.
