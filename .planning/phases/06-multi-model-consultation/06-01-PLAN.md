  <action>
Create `references/consultation-config.md` with:
- Title: "Consultation Configuration"
- Table mapping each of the 9 stages to consultation behavior:
  - intake: none
  - research: routine (deepseek, stage output summary)
  - roadmap: routine (deepseek, roadmap draft)
  - phase-plan: routine (deepseek, plan files)
  - execute: none
  - verify: critical (codex if opted-in, else deepseek)
  - test-gate: none
  - advance: none
  - deploy: critical (codex if opted-in, else deepseek)
- Context size limits: ~2000 chars for routine (DeepSeek), ~4000 chars for critical (Codex)
- Brief rationale section explaining why these stages were chosen

Create `lib/aegis-consult.sh` with these functions:

1. **`get_consultation_type(stage_name)`** — Reads `references/consultation-config.md` and returns "none", "routine", or "critical" for the given stage. Use a bash case statement (not file parsing) for speed and reliability. The config file is the reference doc; the case statement is the implementation.

2. **`consult_sparrow(message, use_codex, timeout_secs)`** — Sends message to Sparrow. Parameters:
   - message: the consultation prompt
   - use_codex: "true" or "false" (default "false")
   - timeout_secs: seconds before timeout (default 60)
   - Uses `${AEGIS_SPARROW_PATH:-/home/ai/scripts/sparrow}` for testability
   - Wraps call with `timeout` command
   - On success: echoes result, returns 0
   - On failure/timeout/unavailable: echoes empty string, returns 0 (NEVER fails the pipeline)
   - Validates result is not an error string (check for "Error"/"error" patterns per `validate_sparrow_result` in aegis-validate.sh)

3. **`build_consultation_context(stage, project)`** — Builds a structured review prompt. Uses python3 to compose:
   - "Review this {stage} output for project {project}:"
   - Reads stage-specific output files from `.aegis/` if they exist
   - Truncates to char limit (2000 for routine, 4000 for critical)
   - Asks reviewer to flag: architectural consistency, missing edge cases, security implications, scope creep
   - Requests "3-5 bullet points" response format

4. **`show_consultation_banner(model, stage, result)`** — Displays formatted consultation result. Match the existing banner style from `show_checkpoint` in aegis-gates.sh:
   - Box-drawing header: "CONSULTATION: {model} Review ({stage})"
   - Result body
   - Divider line at bottom

5. **`read_codex_opt_in()`** — Reads `codex_opted_in` from `.aegis/state.current.json` config section. Returns "true" or "false". Defaults to "false" if field is missing. Uses python3 for JSON parsing.

6. **`set_codex_opt_in(value)`** — Writes `codex_opted_in` to state config. Atomic write (tmp + mv) pattern matching aegis-state.sh.

Script header: `#!/usr/bin/env bash`, `set -euo pipefail`, source `aegis-state.sh` from same directory.

CRITICAL: `consult_sparrow` must NEVER pass `--codex` flag unless `use_codex` parameter is explicitly "true". This enforces the hard rule from CLAUDE.md.
