**01-intake.md** (custom):
- Inputs: User's project description (from conversation), .aegis/state.current.json
- Actions: (1) Ask user to describe their project idea, (2) Extract key requirements, tech preferences, and constraints, (3) Create .planning/ directory structure via `/gsd:new-project` or manually, (4) Write PROJECT.md with extracted info, (5) Create initial REQUIREMENTS.md
- Outputs: .planning/PROJECT.md, .planning/REQUIREMENTS.md
- Completion: Both files exist with non-empty content

**02-research.md** (delegates to GSD):
- Inputs: .aegis/state.current.json, .planning/ROADMAP.md (to determine current phase)
- Actions: (1) Read current phase from roadmap, (2) Invoke `/gsd:research-phase {phase_number}`, (3) Wait for completion
- Outputs: .planning/phases/{phase}/RESEARCH.md
- Completion: RESEARCH.md exists for current phase

**03-roadmap.md** (custom):
- Inputs: .planning/PROJECT.md, .planning/REQUIREMENTS.md
- Actions: (1) Analyze requirements for dependency order, (2) Group into phases (aim for 4-8 phases), (3) Create ROADMAP.md with phase details, success criteria, requirement mapping
- Outputs: .planning/ROADMAP.md
- Completion: ROADMAP.md exists with at least one phase defined

**04-phase-plan.md** (delegates to GSD):
- Inputs: .aegis/state.current.json, .planning/ROADMAP.md
- Actions: (1) Determine current phase, (2) Invoke `/gsd:plan-phase {phase_number}`, (3) Wait for completion
- Outputs: .planning/phases/{phase}/*-PLAN.md files
- Completion: At least one PLAN.md exists for current phase

**05-execute.md** (delegates to GSD):
- Inputs: .aegis/state.current.json, .planning/phases/{phase}/*-PLAN.md
- Actions: (1) Determine current phase, (2) Find next unexecuted plan, (3) Invoke `/gsd:execute-plan {plan_path}`, (4) Repeat for remaining plans in phase
- Outputs: Code files per plan, *-SUMMARY.md per plan
- Completion: All plans in current phase have SUMMARY.md files

**06-verify.md** (delegates to GSD):
- Inputs: .aegis/state.current.json, .planning/phases/{phase}/*-SUMMARY.md
- Actions: (1) Invoke `/gsd:verify-work {phase_number}`, (2) Review verification results
- Outputs: .planning/phases/{phase}/*-VERIFICATION.md
- Completion: VERIFICATION.md exists and all checks pass (or gaps identified)

**07-test-gate.md** (custom):
- Inputs: .aegis/state.current.json, tests/run-all.sh
- Actions: (1) Run `bash tests/run-all.sh`, (2) Capture exit code and output, (3) If tests fail, report which tests failed and signal stage failure, (4) If all pass, signal stage completion
- Outputs: Test results (stdout)
- Completion: All tests pass (exit code 0 from run-all.sh)

**08-advance.md** (custom -- critical stage):
- Inputs: .aegis/state.current.json, .planning/ROADMAP.md, lib/aegis-git.sh
- Actions:
  (1) Source lib/aegis-git.sh
  (2) Determine completed phase number and name from state/roadmap
  (3) Call `tag_phase_completion "$phase_number" "$phase_name"` (GIT-01)
  (4) Mark the phase as complete in ROADMAP.md (check the checkbox)
  (5) Count remaining unchecked phases from ROADMAP.md using python3 (the pattern from 03-RESEARCH.md code examples)
  (6) If remaining_phases > 0: signal advance_stage with remaining count (loops to phase-plan)
  (7) If remaining_phases == 0: signal advance_stage with 0 (proceeds to deploy)
- Outputs: Git tag, updated ROADMAP.md
- Completion: Tag created, roadmap updated, stage advanced correctly

**09-deploy.md** (custom, minimal for v1):
- Inputs: .aegis/state.current.json, .planning/ROADMAP.md
- Actions: (1) Announce that all phases are complete, (2) Summarize what was built (list completed phases from roadmap), (3) Suggest manual deployment steps based on detected stack, (4) Signal pipeline completion
- Outputs: Deployment summary (stdout)
- Completion: Summary displayed, pipeline marked complete

IMPORTANT: Keep each workflow UNDER 100 lines. These are prompt documents Claude follows, not scripts. GSD-delegating stages should be especially lean (just read context, invoke command, validate output).

Do NOT delete stub.md yet -- the orchestrator update in Task 2 will remove the fallback reference.
