  <name>Task 1: Create git operations library and rollback skill</name>
  <files>lib/aegis-git.sh, skills/aegis-rollback.md</files>
  <action>
Create `lib/aegis-git.sh` following the exact patterns from aegis-state.sh and aegis-gates.sh:
- `set -euo pipefail` at top
- `AEGIS_LIB_DIR` pattern for locating sibling libraries
- Source `aegis-state.sh` for state access (same pattern as aegis-gates.sh)

Implement these functions (use the code examples from 03-RESEARCH.md as the starting point):

1. `tag_phase_completion(phase_number, phase_name)`:
   - Creates lightweight git tag `aegis/phase-${phase_number}-${phase_name}`
   - Skips silently if tag already exists (idempotent)
   - Prints "Tagged: {tag_name}" on success

2. `list_phase_tags()`:
   - Lists all `aegis/*` tags sorted
   - Prints one tag per line, or "No aegis tags found." if none

3. `check_rollback_compatibility(target_tag)`:
   - First checks `git status --porcelain` -- if dirty, prints error to stderr and returns 1
   - Then checks `git diff --name-only "$target_tag"..HEAD` for migration/schema file patterns: `*/migrations/*`, `*.sql`, `*/alembic/*`, `*/prisma/*`, `*/knex/*`, `*/sequelize/*`, `*/drizzle/*`
   - Prints "warn-migrations" and the file list to stderr if migration files found
   - Prints "compatible" if no migration files changed
   - Returns 0 in both warn and compatible cases (only returns 1 for dirty tree)

4. `rollback_to_tag(target_tag)`:
   - Verifies tag exists (list available aegis tags if not)
   - Creates new branch `rollback/aegis-phase-N-name-{epoch}` from the tag
   - Updates `.aegis/state.current.json` to reflect rollback position by reading the state from the tag's commit (use `git show {tag}:.aegis/state.current.json` if it exists, otherwise warn that state recovery is unavailable)
   - Prints confirmation message

Create `skills/aegis-rollback.md` following the aegis-launch.md pattern:
- YAML frontmatter with `allowed-tools: ["Bash"]` and description
- $ARGUMENTS is the phase number or tag name (optional -- lists tags if omitted)
- Workflow: source lib/aegis-git.sh, then:
  - If no args: call list_phase_tags()
  - If arg provided: resolve to tag name (if number, convert to `aegis/phase-N-*` pattern via git tag -l), run check_rollback_compatibility(), show results, ask user to confirm if warnings, then call rollback_to_tag()
  </action>

  </verify>
