Then implement `lib/aegis-gates.sh`:

```bash
#!/usr/bin/env bash
# Aegis Pipeline -- Gate evaluation, banners, checkpoints
set -euo pipefail
```

Source `lib/aegis-state.sh` at the top for state access.

Functions to implement:

1. `evaluate_gate(stage_name, yolo_mode)` -- Read the stage's gate object from state.current.json via python3. Based on gate type:
   - "none": echo "pass", return 0
   - "quality" or types containing "quality": check if stage status == "completed". If yes: echo "pass". If no: echo "fail".
   - "approval": if yolo_mode=="true": echo "auto-approved". Else: echo "approval-needed".
   - "external" or types containing "external": always echo "approval-needed" (never skippable).
   - For compound types like "quality,external": evaluate quality first (must pass), then external (always needs approval). If quality fails, return "fail". If quality passes, return "approval-needed".

2. `check_gate_limits(stage_name)` -- Read gate.attempts, gate.max_retries, gate.timeout_seconds, gate.first_attempt_at from state. Compare attempts >= max_retries (if max_retries > 0). Compare elapsed time > timeout_seconds (if timeout_seconds > 0 and first_attempt_at is set). Return "retries-exhausted", "timed-out", or "ok".

3. `record_gate_attempt(stage_name, result, error_msg)` -- Use python3 to update gate.attempts += 1, gate.last_result = result, gate.last_error = error_msg in state.current.json. Use atomic write pattern (tmp + mv).

4. `init_gate_state(stage_name)` -- If gate.first_attempt_at is null, set it to current UTC timestamp. If already set, do nothing. Atomic write.

5. `show_transition_banner(stage_name, stage_index)` -- Format using ui-brand.md pattern. Uppercase stage name (tr), calculate progress percentage (stage_index * 100 / 9), build progress bar (filled/empty blocks). List completed stages. Show "Next up: {next_stage}".

6. `show_checkpoint(checkpoint_type, summary, action_prompt)` -- Format using ui-brand.md checkpoint box pattern. Default action_prompt: 'Type "approved" to advance, or describe issues'.

7. `show_yolo_banner(stage_name)` -- Compact one-liner: "[auto-approved] {stage_name} -- YOLO mode" using the lightning bolt symbol from ui-brand.md.

8. `set_pending_approval(stage_name, pending)` -- Set gate.pending_approval to true/false in state. Used by orchestrator to track approval state across invocations.

All JSON manipulation via python3 (Phase 1 decision). All state writes use atomic tmp+mv pattern.
