Implement `lib/aegis-evidence.sh` with four functions. Follow the exact patterns from research (Pattern 2-5). Use the project-standard approach: bash wrapper calling python3 for JSON/hash operations, atomic tmp+mv writes.

**write_evidence(stage, phase, files_json, requirements_json, stage_specific_json, checks_json):**
- Creates `.aegis/evidence/{stage}-phase-{phase}.json`
- Computes SHA-256 hashes for files in files_json using python3 hashlib
- Reads project and pipeline_id from state.current.json (if exists)
- Stamps AEGIS_POLICY_VERSION (from env var)
- Uses atomic tmp+mv write pattern
- Returns the path to the evidence file

**validate_evidence(stage, phase):**
- Checks evidence file exists (returns "missing" / exit 1 if not)
- Validates all 8 required fields present
- Verifies file hashes match current file content (skip for "file-not-found" hashes)
- Checks requirements_addressed is non-empty
- Returns "valid" (exit 0) or "invalid" (exit 1, errors to stderr)

**query_evidence(requirement_id):**
- Scans all .json files in .aegis/evidence/
- Returns JSON array of matching evidence (file, stage, phase, timestamp, status)
- Returns "not-found" (exit 1) if no evidence references that requirement

**validate_test_requirements(test_output):**
- Counts PASS: lines (reject if 0 = empty suite)
- Extracts [REQ-ID] patterns using regex `\[([A-Z]+-\d+)\]`
- Rejects if no requirement IDs found
- Returns JSON array of sorted unique requirement IDs on success

All python3 code uses ONLY stdlib (json, hashlib, os, glob, re, sys, datetime). No external dependencies.
