1. Add `get_enforcement_mode()` helper function to `lib/aegis-validate.sh`:
   ```bash
   get_enforcement_mode() {
     local stage_name="${1:?get_enforcement_mode requires stage_name}"
     if [[ -z "${AEGIS_POLICY_FILE:-}" ]] || [[ ! -f "${AEGIS_POLICY_FILE:-}" ]]; then
       echo "none"
       return 0
     fi
     python3 -c "
   import json
   with open('${AEGIS_POLICY_FILE}') as f:
       p = json.load(f)
   print(p.get('behavioral_enforcement', {}).get('${stage_name}', 'none'))
   " 2>/dev/null || echo "none"
   }
   ```

2. Replace existing `validate_behavioral_gate()` with stage-aware version:
   - Add second parameter: `local stage_name="${2:-unknown}"`
   - If marker present (grep finds BEHAVIORAL_GATE_CHECK): return 0 (always pass regardless of mode)
   - If marker absent: call `get_enforcement_mode "$stage_name"`
     - "block": stderr "BEHAVIORAL GATE BLOCKED: subagent at stage '{stage_name}' did not output BEHAVIORAL_GATE_CHECK -- mutating actions prevented", return 1
     - "warn": stderr "BEHAVIORAL GATE WARNING: subagent did not output BEHAVIORAL_GATE_CHECK checklist", return 0
     - "none"/default: return 0 silently

3. CRITICAL backward compatibility: `stage_name` defaults to `"unknown"`. When existing code calls `validate_behavioral_gate "$text"` with one arg, the stage is "unknown", which is not in the policy config, so `get_enforcement_mode` returns "none", and the function returns 0 silently. This preserves existing behavior.
